# ESP32-S3 AP配置项目 代码流程图详解

1112

## 项目概述

本项目实现了一个ESP32-S3的WiFi配网系统，支持：
- AP模式：ESP32作为热点，允许用户连接并配置WiFi
- STA模式：ESP32连接到用户指定的WiFi热点
- Web配网页面：通过浏览器配置WiFi信息
- 按键触发配网模式：长按按键进入AP配网模式

---

```
main/
```

### 1.3 按键状态机

```
                    ┌──────────────────────────────────────────────┐
                    │                  状态机                      │
                    └──────────────────────────────────────────────┘

     ┌──────────┐  按下   ┌──────────┐  按下>10ms  ┌──────────┐  按下>1秒   ┌──────────────────┐
     │ RELEASE  │────────▶│  PRESS   │────────────▶│  HOLD    │────────────▶│LONG_PRESS_HOLD  │
     │  (松开)  │         │  (消抖)  │             │ (长按中)  │             │   (等待松手)     │
     └──────────┘         └──────────┘             └──────────┘             └──────────────────┘
          ▲                       │                         │                         │
          │                       │                         │                         │
          │     松开(噪声)        │       松开(<1秒)         │       松开按键            │
          │                       ▼                         ▼                         │
          │                  ┌──────────┐              ┌──────────┐                  │
          └──────────────────│ 短按回调  │◀────────────│ 短按回调  │                  │
                             └──────────┘             └──────────┘                  │
                                                                            ┌────────▼────────┐
                                                                            │     释放        │
                                                                            └─────────────────┘

     ─────────────────────────────────────────────────────────────────────────────────
     状态说明:
     RELEASE         : 释放状态，等待按键按下
     PRESS           : 消抖状态，确认是否为有效按下（10ms消抖）
     HOLD            : 长按状态，计时等待长按触发（1秒）
     LONG_PRESS_HOLD : 长按触发后等待松手状态
     ─────────────────────────────────────────────────────────────────────────────────
```

**说明**：按键采用状态机设计，包含 4 个状态。RELEASE 是初始等待状态；PRESS 是消抖状态，检测到按下后需维持 10ms 才能确认为有效按下，防止噪声干扰；HOLD 是长按状态，按下超过 1 秒触发长按回调；LONG_PRESS_HOLD 是长按后等待松手状态，防止重复触发。短按（按下<1秒松开）会触发短按回调。

### 1.4 定时器扫描回调 button_handle

```
button_handle(arg)  [每5ms执行一次]
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  遍历按键链表                                                   │
│                                                                 │
│  for(btn_info; btn_info; btn_info = btn_info->next)          │
│  {                                                             │
│      gpio_level = btn_info->btn_cfg.getlevel_cb(gpio_num);   │
│                                                                 │
│      switch(btn_info->state)                                  │
│      {                                                        │
│          case BUTTON_RELEASE:                                 │
│              if(gpio_level == active_level)                   │
│              {                                                │
│                  state = BTTON_PRESS;                         │
│                  press_cnt += 5;                              │
│              }                                                │
│              break;                                           │
│                                                                 │
│          case BTTON_PRESS:                                    │
│              if(gpio_level == active_level)                   │
│              {                                                │
│                  press_cnt += 5;                              │
│                  if(press_cnt >= 10)  // 消抖10ms            │
│                      state = BUTTON_HOLD;                     │
│              }                                                │
│              else                                             │
│              {                                                │
│                  state = BUTTON_RELEASE;  // 噪声            │
│                  press_cnt = 0;                              │
│              }                                                │
│              break;                                           │
│                                                                 │
│          case BUTTON_HOLD:                                    │
│              if(gpio_level == active_level)                   │
│              {                                                │
│                  press_cnt += 5;                              │
│                  if(press_cnt >= long_press_time)             │
│                  {                                            │
│                      long_press_cb(gpio_num);  // 长按回调    │
│                      state = BUTTON_LONG_PRESS_HOLD;         │
│                  }                                            │
│              }                                                │
│              else                                             │
│              {                                                │
│                  short_press_cb(gpio_num);  // 短按回调       │
│                  state = BUTTON_RELEASE;                      │
│                  press_cnt = 0;                              │
│              }                                                │
│              break;                                           │
│                                                                 │
│          case BUTTON_LONG_PRESS_HOLD:                         │
│              if(gpio_level != active_level)                    │
│              {                                                │
│                  state = BUTTON_RELEASE;                      │
│                  press_cnt = 0;                              │
│              }                                                │
│              break;                                           │
│      }                                                        │
│  }                                                            │
└─────────────────────────────────────────────────────────────────┘
```

**说明**：此函数由 5ms 定时器周期性调用，负责扫描所有注册的按键并更新其状态。每个按键根据当前状态进行不同处理：RELEASE 状态检测到按下则转入 PRESS；PRESS 状态维持 10ms（2次扫描）则确认有效按下转入 HOLD；HOLD 状态若按下超过 1000ms 则触发长按回调并转入 LONG_PRESS_HOLD，若松开则触发短按回调；LONG_PRESS_HOLD 状态等待按键释放后恢复 RELEASE。press_cnt 每次加 5 表示经过了一个扫描周期。

### 1.5 XL9555 中断处理

```
硬件中断触发 (GPIO17下降沿)
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  XL9555_INT_Handle()  [中断处理程序]                            │
│                                                                 │
│  xEventGroupSetBitsFromISR(XL9555_IO_Event, XL9555_ISR_BIT)    │
│  portYIELD_FROM_ISR()                                          │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  XL9555_Task()  [FreeRTOS任务]                                  │
│                                                                 │
│  while(1)                                                      │
│  {                                                             │
│      等待事件组: xEventGroupWaitBits(...)                       │
│                                                                 │
│      esp_rom_delay_us(1000);  // 延时消抖                       │
│                                                                 │
│      if(gpio_get_level(INT) != 0)  // 干扰，跳过               │
│          continue;                                              │
│                                                                 │
│      XL9555_Read_Word(0x00, &Input);  // 读取IO状态            │
│                                                                 │
│      遍历16个引脚:                                              │
│      for(i=0; i<16; i++)                                       │
│      {                                                         │
│          Last_value = Last_Input & (1<<i);                     │
│          value = Input & (1<<i);                               │
│          if(value != Last_value)                               │
│          {                                                     │
│              XL9555_IO_CallBack(1<<i, value);  // 回调通知     │
│          }                                                     │
│      }                                                         │
│                                                                 │
│      Last_Input = Input;  // 更新状态                           │
│  }                                                             │
└─────────────────────────────────────────────────────────────────┘
```

**说明**：XL9555 扩展芯片的中断处理采用中断+任务的分层设计。当芯片引脚电平变化时，GPIO17 产生下降沿触发中断，中断处理程序设置事件组位通知任务。XL9555_Task 任务等待事件后，延时 1ms 消抖，然后读取芯片的 16 位输入端口状态。通过与上次状态对比，检测哪些引脚发生了变化，并调用回调函数通知上层应用。这种设计避免了中断中处理 I2C 通信，保证了中断处理的高效性。

### 3.3 AP_WIFI_Task WiFi连接任务

```
AP_WIFI_Task(param)
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  while(1)                                                      │
│  {                                                             │
│      ev = xEventGroupWaitBits(                                 │
│          APcfg_ev,           // 事件组                         │
│          APCFG_BIT,          // 等待此位                        │
│          pdTRUE,             // 清除位                          │

│  }                                                             │

```

**说明**：此任务是配网逻辑的核心，一直在后台运行等待配网完成事件。任务使用事件组等待机制阻塞 10 秒，当 WebSocket 收到用户输入的 WiFi 凭据并设置 APCFG_BIT 事件位时，任务被唤醒。首先停止 Web 服务器（用户已不需要配网页面），然后调用 WIFI_manager_connect 连接用户指定的 WiFi。连接成功后 ESP32 将自动断开 AP 模式，切换到纯 STA 模式。

```

         │
```

## 重要注意事项

### 内存管理要点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              内存管理                                        │
└─────────────────────────────────────────────────────────────────────────────┘

1. Scan_Task中的AP列表
   ┌─────────────────────────────────────────────────────────────────┐
   │  esp_wifi_scan_get_ap_num(&AP_Count);  // 先获取数量           │
   │  AP_List = malloc(sizeof * AP_List * AP_Count);  // 分配内存  │
   │  esp_wifi_scan_get_ap_records(&AP_Count, AP_List);            │
   │  callback(AP_Count, AP_List);  // 使用                           │
   │  free(AP_List);  // 释放内存                                    │
   └─────────────────────────────────────────────────────────────────┘

2. cJSON使用
   ┌─────────────────────────────────────────────────────────────────┐
   │  char *Data = cJSON_Print(root);  // 分配内存                   │
   │  _Web_WS_Send((uint8_t*)Data, strlen(Data));                   │
   │  cJSON_free(Data);  // 释放JSON字符串                          │
   │  cJSON_Delete(root);  // 释放JSON对象                          │
   └─────────────────────────────────────────────────────────────────┘

3. WebSocket数据接收
   ┌─────────────────────────────────────────────────────────────────┐
   │  httpd_ws_recv_frame(req, &packet, 0);  // 先获取长度          │
   │  char *buf = malloc(packet.len + 1);  // 分配                  │
   │  packet.payload = buf;                                          │
   │  httpd_ws_recv_frame(req, &packet, packet.len);  // 再获取数据 │
   │  ws_receive_callback(packet.payload, packet.len);              │
   │  free(buf);  // 释放                                           │
   └─────────────────────────────────────────────────────────────────┘
```

**说明**：ESP32 内存有限，必须妥善管理内存分配和释放。扫描任务中，先获取 AP 数量再分配内存，避免内存不足时崩溃。cJSON_Print 分配内存用于 JSON 字符串转换，必须用 cJSON_free 释放（注意不是 free）。WebSocket 数据接收分两步：先获取长度，再分配内存接收数据，最后释放缓冲区。

### 线程安全要点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              线程安全                                        │
└─────────────────────────────────────────────────────────────────────────────┘

1. WebSocket回调上下文
   ┌─────────────────────────────────────────────────────────────────┐
   │  ws_receive_handle() 在WebSocket任务上下文中执行                │
   │                                                              │
   │  ❌ 不能直接调用 esp_wifi_connect() (阻塞操作)                  │
   │  ❌ 不能直接调用 esp_wifi_stop()                                │
   │                                                              │
   │  ✅ 应该使用事件组通知WiFi任务执行实际操作                      │
   │     xEventGroupSetBits(APcfg_ev, APCFG_BIT);                  │
   └─────────────────────────────────────────────────────────────────┘

2. 扫描信号量保护
   ┌─────────────────────────────────────────────────────────────────┐
   │  xSemaphoreTake(Scan_sem, 0);  // 获取信号量                    │
   │  if(失败) return ESP_ERR_INVALID_STATE;  // 防止重复扫描       │
   │  ... 执行扫描 ...                                              │
   │  xSemaphoreGive(Scan_sem);  // 扫描完成后释放                  │
   └─────────────────────────────────────────────────────────────────┘
```

**说明**：ESP32 使用 FreeRTOS 操作系统，多任务间需要考虑线程安全。WebSocket 回调运行在 HTTP 服务器任务上下文，不适合执行耗时的 WiFi 操作，否则会阻塞服务器响应。正确做法是设置事件组标志，让专门的任务执行实际连接。扫描信号量确保同一时间只有一个扫描任务运行，防止资源冲突。
